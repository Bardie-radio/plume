# Plume MVP v0.1 scope

Ship a usable web UI against Kithara + Bes (+ Magpie for play). UI stack: [03-ui-stack.md](../03-ui-stack.md). Phases: [implementation-plan.md](implementation-plan.md).

## In scope

- `/`, `/control/{slug}`, `/player/{slug}` behind the edge
- Razor SSR + Vue CSR widgets + Tailwind / shadcn-vue
- BFF session (httpOnly cookie; no JWT in browser)
- Strip Identity / EF / SQLite / Bootstrap scaffold
- Discovery-driven login (Bes `form_schema`)
- Authenticated Struna list/create; access modes on create (**no** encode-mode UI)
- Control desk: queue, search, transport, compact now-playing; **poll**
- Player surface: prominent now-playing; guest code entry → exchange → guest JWT
- Optional listen (audio **off by default**)
- Client Register + OTLP `bardie.plume`

## Out of scope

- `/listen` path
- PWA / themes
- Browser→Kithara SSE/WS (post-MVP; Kithara backlog)
- Calling auth adapters or source modules directly
- Plume-owned user/stream database
- Browser JWT / Plume SignalR
- Account-linking UI
- Beak/Cauda surfaces
- Encode-mode on create (Kithara may still have encode-alive; Plume does not expose a create toggle)

## Implementation checklist

- [ ] Scaffold teardown; Razor host + Tailwind (+ Vite)
- [ ] BFF cookie session + server-side token store; `/bff/*` → Kithara Bearer
- [ ] Edge-ready `/`, `/control/{slug}`, `/player/{slug}`
- [ ] Discovery + authenticate/callback via BFF
- [ ] Struna home: list/create; links to control + player
- [ ] Control desk widgets + poll + BFF mutations
- [ ] Player surface + guest exchange UX
- [ ] Optional `/stream` listen (token query when needed)
- [ ] OTLP + Register as client module
- [ ] Compose as `plume` + join secret
