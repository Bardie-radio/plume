# Plume MVP v0.1 scope

Ship a usable web UI against Kithara + Bes (+ Magpie for play).

## In scope

- `/` and `/player/{slug}` behind the edge
- Discovery-driven login (Bes `form_schema`)
- Authenticated Struna list/create; access modes + encode mode on create
- Player page: now-playing, queue, play/quickplay, pause, skip, search/quicksearch, delete
- Guest code entry → exchange → guest JWT for protected control
- Optional listen (player off by default)
- OTLP `bardie.plume`

## Out of scope

- PWA
- Calling auth adapters or source modules directly
- Plume-owned user/stream database
- Account-linking UI
- Beak/Cauda surfaces

## Implementation checklist

- [ ] Edge-ready app for `/` and `/player/{slug}`
- [ ] Discovery + authenticate/callback + token storage
- [ ] Struna CRUD/control against rest-api sketch
- [ ] Guest exchange UX
- [ ] Optional `/stream` listen (token query when needed)
- [ ] OTLP
- [ ] Compose as `plume` + join secret if registering as client module
